The effect of Packet Loss (try reloading!)

Here, we try loading many (90) small files over a lossy (20% packet loss) and high-RTT (500ms RTT) connection. TCP has the problem of head-of-line blocking, which makes QUIC advantageous in this setup. Note that currently our QUIC implementation of loss detection is very limited, so TCP might (still) outperform us.
